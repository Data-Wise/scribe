    Problem Statement

    Currently, icon mode (48px collapsed) and expanded modes (Compact/Card) are separate systems:

    • ‚ö†Ô∏è Limited Scope: Only Projects can expand; Inbox and Smart Folders stuck in icon-only mode
    • ‚ùå No Persistence: Mode doesn't persist across collapse/expand cycles
    • ‚ùå No Visual Feedback: Users don't know which mode will restore on expand
    • ‚ö†Ô∏è Inconsistent UX: Different behaviors for different item types

    Key Insights from Q&A

    Current Behavior (from questions 1-4)

    1. Expand only for projects - Inbox and Smart Folders can't expand currently
    2. User wants unified behavior - All items should expand to same mode
    3. Always Compact default - Compact is the intended default expanded state
    4. Preset controls mode - Width preset (narrow/medium/wide) determines expand target

    Expansion & Triggers (from questions 5-8)

    1. Add expand to Inbox AND Smart Folders - Full parity across all items
    2. Click + keyboard triggers - Single click OR ‚åò‚áß[ expands
    3. Settings location - Appearance ‚Ä∫ Sidebar Settings
    4. Dual persistence - Settings for preference, localStorage for state

    Cycle & Interaction (from questions 9-12)

    1. Unified expansion - All items expand to same mode (not independent)
    2. Three-way cycle - Compact ‚Üí Card ‚Üí Icon (preset-aware)
    3. Preset sets mode - narrow=Compact, wide=Card
    4. Animation preview on hover - 300ms delay, then show expand target

    Priority & Logic (from questions 13-16)

    1. Preset respects cycle - narrow: C‚ÜíI, medium: C‚ÜíI, wide: C‚ÜíW‚ÜíI
    2. System tooltip delay - Honors OS tooltip timing (300-700ms)
    3. "Remember sidebar mode on collapse" - Setting name
    4. Priority: Setting ON: last mode | Setting OFF: preset

    Manual Resize (from questions 17-20)

    1. Prompt to update preset - "Update preset to this width?"
    2. Immediate preset switch - Changing preset while expanded ‚Üí instant resize + mode change
    3. Each mode has width - Compact remembers 240px, Card remembers 320px separately
    4. System tooltip delay - Use OS preference for hover timing

    Empty States (from questions 21-24)

    1. Empty Inbox action - "Inbox is empty" + Quick Capture button
    2. Empty Smart Folder - "No [type] projects" + Create Project button
    3. Prompt on drag end - Show "Update preset?" immediately after resize
    4. Migration: Default ON - All users (new + existing) get "remember mode" enabled

    Keyboard & Accessibility (from questions 25-28)

    1. ‚åò‚áß[ cycles only - No separate shortcuts for specific modes
    2. User choice - Settings override for reduced-motion (not auto-detect)
    3. Works immediately - Preview animation on first hover (no training)
    4. Debounce 200ms - Prevent rapid clicking spam

    Polish & Testing (from questions 29-32)

    1. No multi-window sync - Each window has independent mode state
    2. ActivityBar indicator - Show "Compact Mode" or "Card Mode" text
    3. "Always update" checkbox - Remember user's preset update preference
    4. Comprehensive E2E tests - Persistence, presets, cycles, animation timing
